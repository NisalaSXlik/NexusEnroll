Software Architecture (SCS 2303) — Assignment 3

1. Introduction

The objective of this assignment is to demonstrate the ability to translate business requirements into a maintainable and scalable software design by applying software design principles, design patterns and architectural patterns.

Key expectations:
- Translate requirements into a technical design (features, data model, interactions).
- Apply SOLID, DRY, KISS and other design principles and justify choices.
- Produce a maintainable and scalable design with error-handling and validation.

2. Duration and Team Composition

- Duration: 4 weeks (Release date: 06 August 2025; Deadline: 06 September 2025).
- Team size: 8–9 students per team.

3. Submission Guidelines

Required artefacts:
1. A software design analysis document containing overall architecture, UML diagrams, and design pattern usage.
2. Source code implementing the design (allowed languages: C++, C#, Java, Python).
3. A screencast video (≤10 minutes) demonstrating core functionality and interaction with the business tier.
4. A single compressed archive containing documentation, screencast, and source code uploaded to the VLE before the deadline.

Notes:
- The screencast should focus on the business logic tier and show user stories (student, faculty, admin flows).
- Ensure code is well-commented and includes instructions for running the demo or tests.
