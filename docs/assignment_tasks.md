Assignment Outline & Tasks — NexusEnroll

Part A — Architectural Design (30%)
1. Select an architectural pattern (Microservices, SOA, or 3-Tier) and justify the choice with respect to scalability, maintainability and integration.
2. Provide a clear architecture diagram labeling layers, services and communication pathways.
3. Describe each major component and its responsibilities.

Part B — Detailed Design & Implementation (70%)
1. Identify core features for Student, Faculty and Administrator modules.
2. Apply a minimum of three distinct OO design patterns; identify and justify pattern choices and locations.
3. Produce UML diagrams: Activity, Sequence (or Collaboration), State, Class, and any other relevant models.
4. Ensure design follows software design principles (encapsulation, SOLID, composition over inheritance). Provide concrete examples for each principle.
5. Implement a small, runnable proof-of-concept application demonstrating the core modules and patterns. Include a simple main/test harness.
6. Document the code, diagrams and pattern usage in the submission report.

Assessment criteria summary:
- Architectural choice & justification
- Architectural diagram & description
- Design pattern application
- Implementation & code quality
- Documentation completeness
